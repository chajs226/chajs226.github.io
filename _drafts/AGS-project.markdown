## 업무협의 내용 (2019.07.09)
### AGS 업무 파악 내용
- 평가문항, 부서, 점수 산출은 매년 달라질 수 있음.
- 평가 세부사항 검토 및 확정 절차 존재
- 년초에 코딩틀을 전체 검토함
- 팀별 평가 / 진료과별 평가 / 직급별 평가 / 집중평가 - 상반기 1회, 하반기 1회
- 평가결과 입력
    - 평가 세부사항 입력. 전체중에 충족건수가 몇건인지 입력, 상세 내용, 개선필요사항, 사유 입력(세부사항은 별도의 시트에 기술)
    - 평가결과에 건수가 들어가기도 하고, 유/무가 들어가기도 함.
    - 점수 산출방법도 문항마다 다름.
- 엑셀에 작업을 하다보니, 오류가 날 수 밖에 없는데, 성과 평가에 활용되기 떄문에 수검부서는 민감함. 각 챕터 담당자들이 본인의 데이터를 검수하는 작업이 약 3주간 소요
- 평가 검토가 끝나면, 진료과별 보고서 산출

### 주요요구사항
- 평가결과 검증에 사람 인력이 많이 들어가기 때문에, 전산에서 걸러줬으면 좋겠음
    - 잘못된 입력된 수치
    - 부서의 대상항목이 아닌데, 측정 항목이 잘못들어가는 경우
    - 전체 코딩툴의 취합본과 개별 입력한 데이터가 다른 경우
    - 부서에 갈 수 없는 문항이 부서에 연결되어 있는 경우
- 보고서를 작성하는데 만드는데 많은 시간이 소요되므로 이를 개선
- 다양한 측면의 보고서가 보여지면 좋겠다(경영진), 데이터 입력오류를 줄이고 싶다(실무진)
- 평가 항목의 분류가 합쳐지고 찢어질 수 가 있고, 큰 항목에서의 지난 3년간의 트랜드를 볼 수 있어야 함 (평가 항목에 대한 히스토리와 맵핑관계가 필요할 수 있음)
- 평가대상부서의 진료과도 인사조직, 진료 조직과 다르기 때문에, AGS를 위한 별도의 조직 맵핑 정보를 가져가야 함. (이것도 년도별로 정보관리가 되어야 함)
- **년중에 부서가 없어지거나 변경되는 경우, AGS평가 정보가 이전 부서의 기준으로 유지되어야 하는지, 변경된 부서로 옮겨가야 하는지 정의 필요 (히스토리컬하게 가야 되는지의 중요 요인)**
- 인턴은 별도의 코딩툴이 있음(인턴은 계속 진료과를 돌기 때문에)
- 직종을 구분으로 평가하는 case도 있음 (이송직원, 청소직원, 린넨 - 원무팀을 통해서 전달이되기는 함)
