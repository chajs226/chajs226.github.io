# 현황
- 2.0 에서는 불출과 상관없이 수행기록함
- 마약류 관리의 대상이 되는 마약의 정의 : “중점관리 마약류”(pofol 포함), 일반향정은 별도 프로세스로 진행(현재 비품인계만 있음. 예: midazolam)
- 회복실 마약사용이 많으므로 검사실 및 지원부서의 마약 사용도 고려 필요
- 크게 병동, 지원부서, 수술실의 마약사용으로 나누어서 구체화

# 기능 요건
## 1. 마약 인증
### 미인증 마약 불출건 소급 건
- 나의 할일(To-Do)에 미인증 데이터 추가 적재후 처리
    - 1주 경과 후, 미인증건에 대해서 의사에게 별도 공지
- 일부 vald 처리 오류건 데이터 작업(IT 서비스팀, 수술장 PRN 29건)
-  프로그램 수정 (6/17)
    - 미인증 마약이 불출되지 않도록 확인 메시지 팝업되도록 수정. (프로그램 수정 이후로는 미인증 마약 불출 건 발생하지 않음)

### 지원부서 추가처방 전자 인증
- 의사 외 사용자가 '지원부서 추가처방'이나 '검사간호기록'으로 입력한 경우 의사 후인증 처리가 필요하기 때문에, '지원부서 추가처방' 화면에서 reminder 팝업으로 후인증 처리를 유도함 (로그인의사&근무장소 같은경우 -> 로그인 의사 기준으로 변경)

### 마약처방전 발행 의사 표기
- 처방의 vs 인증의 처리 방안 IT 서비스팀 검토

## 2. 마약 교환(맞바꿈)

### 마약교환이 가능한 범위 및 조건
- 시범 적용대상: 입원환자(병동/ICU)
- 교환 대상: 간호사가 Acting한 마약(PRN, B처방)
- 교환 조건: 환자/내원/약배송장소/총 조제량이 모두 동일한 경우 
- 교환 신청: 간호사
- 교환 승인: 약사
    - 전산 자동으로 처리(배치 주기 검토, IT서비스팀)하고, 추후 불출 약사가 확인하여 최종 승인 처리
    - 승인(약국) 이후에는 교환 취소(간호)를 할 수 없음
- 의사 반납 후인증 필요

### 추가 고려사항
- 처방횟수가 1이 아닌 경우, 회차별, 부분 불출 등 고려
- PRN 회차별 acting 한 마약에 대해서만 우선 마약교환 고려
- PRN 처방 Acting 횟수 유효성 체크 프로세스 검토 필요
    - Acting 취소 후, 약 반납없이 추가 Acting 하여 원처방의 투약 횟수를 초과하는 케이스
- 집계 전 교환은 지양해야 하지 않을까 (검사실은 집계가 미리 되지 않음)
- 추후의 투약보고 고려 필요

## 3. 마약장부 전산화
- 세부 논의 진행 중

## 의사결정회의
- 수기처방전 출력 권한 제한 해제하기로 함 ('19.7.9)

# 기타

## 미인증 마약 불출 건 소급 방안
- 소급 처리 방안에 대한 의사결정을 위해 데이터 확인 필요
    - 총 430건
    - 미인증 마약 처방지시의사 : 총 103명
    - 입력 프로그램
        - PRN액팅(통합처방화면에서 D/C된 약 vald 살리는 작업 수정 요청)
        - 그외(건진, CP, 입원처방, 검사간호기록, 지원부서추가처방, 통합처방, 원무추가)
    - 430건 인증할 수 있는 별도 프로그램 개발 검토 중
